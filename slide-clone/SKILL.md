## 개요

기존 슬라이드 디자인을 분석해서 **완전한 시스템 프롬프트로 변환**합니다.

사용자가 다음과 같이 요청하면:
```
@/path/to/deck.pdf 이걸로 프롬프트 만들어줘
```

스킬이 자동으로:
1. PDF/PowerPoint 읽기 (최대 15페이지)
2. 색상, 폰트, 레이아웃, 데코레이션 추출
3. 슬라이드 레이아웃 9가지 식별
4. 콘텐츠 규칙 분석
5. 완전한 시스템 프롬프트 생성 (§1~§10)
6. `~/Desktop/{파일명}-prompt.md` 자동 저장

| 항목 | 형식 |
|---|---|
| **입력** | PDF 또는 PowerPoint (.pptx, .ppt) |

## 작동 방식

### Step 1: 파일 분석
- 처음 10~15페이지에서 색상, 폰트, 레이아웃, 데코레이션 추출
- 각 슬라이드 타입 식별 (표지, 섹션+내용, 비교표, 차트, 닫기 등)
- 콘텐츠 패턴 (제목 톤, 강조 방식, 숫자 포맷) 기록

### Step 2: 시스템 프롬프트 생성
- §1: 역할 (역할 정의, 기본값 PPTX로 설정)
- §2: 디자인 시스템 (색상 토큰, 타이포, 그리드, 데코)
- §3: 슬라이드 레이아웃 카탈로그
- §4: 콘텐츠 작성 규칙
- §5~§10: 섹션별 심화 가이드

### Step 3: Desktop 저장
```
~/Desktop/{원본파일명}-prompt.md
```

예:
```
[IR Pitch Deck] Liner KR.pdf
→ [IR Pitch Deck] Liner KR-prompt.md
```

## 추출 기준

### 색상 팔레트
모든 색상을 hex 값으로 식별:
```
--primary:     #XXXXXX  (주요 용도)
--secondary:   #XXXXXX  (보조 용도)
--accent:      #XXXXXX  (강조)
```

### 타이포그래피
- 제목: font-family, weight, letter-spacing, line-height
- 본문: 동일
- 라벨: weight, spacing 추가 규칙

### 레이아웃
- 슬라이드 비율 (16:9, 4:3 등)
- 기본 패딩 / 마진
- 그리드 단위 (px, cqi, em 등)

### 데코레이션
- 표지 배경 / 로고 위치
- 섹션 라벨 (색상, 크기, 위치)
- 카드 / 테이블 스타일
- 특수 요소 (모서리 괄호, 선, 패턴 등)

---

## 슬라이드 레이아웃 분류

파일의 슬라이드를 다음 9가지로 분류:

| # | 이름 | 용도 |
|---|---|---|
| 1 | 표지 | 회사명, 로고, 날짜 |
| 2 | 섹션+내용 | 제목 + 카드/다이어그램 |
| 3 | 미션/비전 | 인용구 + 사례 2개 |
| 4 | 문제 정의 | 제목 + 타임라인 테이블 |
| 5 | 제품 아키텍처 | 다이어그램 (3단계) |
| 6 | 비교 테이블 | 자사 vs 경쟁사 비교 |
| 7 | 가격/성과 카드 | 3개 카드 또는 4개 이상 |
| 8 | 막대 차트 | 수평/수직 막대 |
| 9 | 마무리 | 핵심 메시지 + 다음 단계 |

## 콘텐츠 규칙 분석

### 제목 / 부제
- 길이 (한 줄 vs 두 줄)
- 톤 (평서문, 의문문, 명령)
- 종결 표현 ("~합니다", "~입니다" 등)

### 강조
- 굵음만 사용?
- 배경색 사용?
- 색상 조합 규칙?

### 카드 / 박스
- 제목 길이
- 본문 길이
- 테두리 / 그림자 스타일

### 테이블
- 헤더 배경색
- 강조 셀 (색상 또는 테두리)
- 숫자 정렬

### 다이어그램
- 계층 (상/중/하)
- 배경색 구분
- 텍스트 색상 (검은색 vs 흰색)
- 화살표 / 연결선 스타일

### 한국어 규칙 (상세는 references/content-rules.md)
- em dash 사용 여부
- 번역투 / 직역체
- 주술 구조 정합
- 명사구 깊이
- 종결 표현 일관성

### 숫자 / 단위
- 천 단위 콤마 (1,420 vs 1420)
- 소수점 자릿수
- 단위 위치 (12.3%, 2.4x, $120)
- 약어 표기 (KPI vs 케이피아이)
- 시점 표기 (2026년 1분기 vs 2026 Q1)

## 추가 주의사항

1. **정확성**: 추출한 색상/폰트/레이아웃은 정확해야 합니다. 추측하지 마세요.
2. **완결성**: §1~§10 모두 포함. 정보 부족하면 "정보 부족" 사유 명시.
3. **기본값 PPTX**: 생성 프롬프트는 항상 PPTX를 기본값으로 설정 ("사용자가 명시하지 않으면 PPTX 생성").
4. **한국어**: 모든 섹션 한국어로 작성 (영어 예제 제외).
5. **예제**: 각 섹션에 Before/After 또는 HTML 코드 블록 포함.
