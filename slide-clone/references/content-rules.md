# 콘텐츠 규칙 상세 가이드

## 4.1 제목 (Title / Section Heading)

### 길이
- 한 줄 또는 두 줄
- 한국어 기준: 20~40자

**예:**
- ✅ "인류의 정보 탐색 방식은 계속해서 진화해왔습니다"
- ❌ "정보 탐색의 진화"

### 톤
- 평서문 (주로)
- 의문문 (드물게)
- 감탄문 (거의 없음)
- 명령 (거의 없음)

### 종결 표현

| 표현 | 빈도 | 톤 |
|---|---|---|
| ~합니다 | 70% | 중립적, 정중 |
| ~입니다 | 20% | 단정적 |
| ~됩니다 | 10% | 수동적 |

**기준:** 한 슬라이드 또는 한 섹션 내에서 종결 표현을 통일합니다.

---

## 4.2 부제 (Subtitle / Description)

### 관계
- 제목이 "무엇이 사실인가"라면
- 부제는 "그래서 무엇이 중요한가"

**예:**
- 제목: "인류의 정보 탐색 방식은 계속해서 진화해왔습니다"
- 부제: "이 변화에 대응하지 못하면 경쟁력을 잃을 수 있습니다"

### 길이
- 한 문장
- 한 줄~두 줄

---

## 4.3 강조 (Emphasis)

### 선택 규칙

| 방식 | 사용 시 | 예 |
|---|---|---|
| **굵음** | 숫자, 개념, 키워드 | **50ms**, **4가지 구조** |
| 배경색 | 중요 구절 | 라임색 배경: "매우 빠른" |
| 색상 + 굵음 | 거의 사용 안 함 | ❌ 피하기 |

**기준:** 굵음과 배경색을 함께 사용하지 않습니다.

### 배경색 선택

| 색상 | 강도 | 용도 |
|---|---|---|
| --lime-pale (#E8F0BF) | 약함 | 보조 강조, 부드러운 강조 |
| --lime-accent (#C4D96F) | 강함 | 핵심 강조, 중요 데이터 |
| --highlight (#D4472A) | 극강 | 경쟁사 비교 (자사 우위) |

## 4.4 카드 본문

### 제목
- 명사구
- 6~12자
- 예: "정보 탐색 혁신", "기술 우위"

### 본문
- 1~2 문장
- 각 문장 25~50자
- 종결: ~합니다 / ~입니다

**예:**
```
카드 제목: 정보 탐색 혁신

카드 본문:
인류는 수십 년에 걸쳐 정보를 찾는 방식을 
계속 바꾸고 있습니다. 검색엔진, SNS, AI 기반 
검색 등의 등장이 이를 증명합니다.
```

### 통계 (카드 내)
```
숫자 + 단위 + 라벨

50ms
검색 속도 (약화 텍스트, 0.9em)
```

## 4.5 테이블

### 구조

```html
<thead>
  <tr style="background: --lime-pale;">
    <th>지표</th>
    <th>라이너</th>
    <th>경쟁사</th>
  </tr>
</thead>
<tbody>
  <tr>
    <td>검색 속도</td>
    <td style="border: 2px solid --highlight;"><strong>50ms</strong></td>
    <td>150ms</td>
  </tr>
</tbody>
```

### 헤더
- 배경: --lime-pale
- weight: 600
- 색상: 검은색 (기본)

### 셀
- 기본 배경: --paper (흰색)
- 테두리: --line (회색)
- 강조 셀 (자사/우위):
  - 빨간 테두리 (--highlight) 또는
  - 라임색 배경 (--lime-pale)

### 통화 / 숫자
- 우측 정렬
- weight: 700 (큰 숫자)
- weight: 600 (차트 내 숫자)

### 범례
- 테이블 하단
- font-size: 0.85em
- 색상: --muted

## 4.6 다이어그램 / 아키텍처

### 3단 구조 (가장 흔함)

```
┌─────────────────────┐
│   Consumer (밝음)    │  ← 배경: 흰색 또는 연한 색
└─────────────────────┘
           ↓
┌─────────────────────┐
│   Product (라임)     │  ← 배경: --lime-accent
└─────────────────────┘
           ↓
┌─────────────────────┐
│ Technology (짙음)    │  ← 배경: --forest-dark
└─────────────────────┘
```

### 색상 규칙
- 상단: 밝은 색 (흰색) 또는 --lime-pale
- 중간: --lime-accent
- 하단: --forest-dark

### 텍스트 색상
- 밝은 배경 위: 검은색 (#000000)
- 어두운 배경 위: 흰색 (#FFFFFF)

### 화살표 / 연결선
- 색상: --line (회색) 또는 --forest-dark (짙은 녹색)
- 굵기: 2px

## 4.7 한국어 표기 원칙

### Em dash 금지
❌ "이는 ~ — 즉, ~"
✅ "이는 ~ (즉, ~)"
✅ "이는 ~: 즉, ~"

### 번역투 제거

| 번역투 | 개선 |
|---|---|
| ~에 대해 ~를 가지다 | ~를 가지다 |
| 이는 ~을 의미한다 | ~이다, ~라는 뜻이다 |
| ~에 있어서 | ~에서 |
| ~의 실현 | AI를 도입한다 |
| 혁신성, 디지털화 (누적) | 최대 2개 |

### 주술 구조 정합

❌ "이 데이터는 우리의 성공을 보여준다"
(무생물 주어가 직접 동사)

✅ "이 데이터를 보면 우리가 성공했다"
(사람/행위 주어)

### 명사구 깊이
- 3단 이상 소유격 연결 금지
  - ❌ "회사의 제품의 기능의 특징"
  - ✅ "회사 제품의 기능"

- 형용사 4개 이상 누적 금지
  - ❌ "빠르고 정확하고 안전하고 효율적인 솔루션"
  - ✅ "빠르고 정확한 솔루션"

### 종결 일관성
한 슬라이드 내에서:
- "~합니다" 또는 "~입니다"로 통일
- 섞지 않기

## 4.8 숫자 · 단위 · 약어

### 천 단위 콤마
- 4자리 이상: 콤마 찍기
  - ✅ 1,420명, 12,300억 원
  - ❌ 1420명, 12300억 원
- 예외: 연도, 페이지 번호, 버전
  - ✅ 2026, 페이지 123, v1.0

### 소수점 자릿수
- 기본: 정수 또는 소수 1자리
  - ✅ +12.3%, 2.4x, 42
- 예외: 정밀도 필요할 때만 2자리
  - ✅ ±0.15% (금융 관련)

### 단위 위치

| 단위 | 위치 | 예 |
|---|---|---|
| % | 숫자 바로 뒤 | +12.3% |
| x | 숫자 바로 뒤 | 2.4x |
| bp | 숫자 바로 뒤 | 150bp |
| 통화 | 숫자 앞 | $142, ₩1,200 |

### 방향 부호
```
증가: +3.1%p (percentage point)
감소: -68%
```

### 단위 일관성
한 슬라이드 안에서 같은 지표는 같은 단위:
- ✅ "2026년 1분기", "2026년 2분기"
- ❌ "2026 Q1", "2026년 2분기" (혼동)

### 영문 약어

| 약어 | 표기 | 규칙 |
|---|---|---|
| KPI | KPI | 영문 대문자 (한글 음차 금지) |
| ROI | ROI | ~ |
| CEO | CEO | ~ |
| M&A | M&A | ~ |

**첫 등장 규칙:**
```
KPI(핵심 성과 지표)는 ~합니다.
  ↑ 괄호로 풀이, 이후 반복 안 함
```

### 고유명사 · 브랜드
- 사용자 표기 그대로 따르기
- 예: "Liner AI Search", "OpenAI", "Anthropic"

### 시점 표기

| 형식 | 선택 |
|---|---|
| 분기 | "2026년 1분기" 또는 "2026 Q1" 중 하나로 통일 |
| 월 | "2026년 4월" 또는 "2026.04" 중 하나로 통일 |
| 일 | "2026.04.20" 또는 "2026년 4월 20일" 중 하나로 통일 |

한 슬라이드 내 일관성 유지.
